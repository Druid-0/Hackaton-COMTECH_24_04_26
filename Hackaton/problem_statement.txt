Формулировка задачи

Контекст:
У вас есть baseline кредитного скоринга в файле baseline.ipynb.
Нужно на его основе сделать полноценное решение с пользовательским интерфейсом.

Данные:
- Один датасет: data/credit_scoring_dataset.csv
- 500 записей (кредитные заявки)
- 8 признаков
- Цель: target (1 = default / плохой заемщик, 0 = non-default / хороший заемщик)

Основная задача:
Сделать приложение, которое принимает на вход значения признаков новой заявки
и возвращает прогноз:
- default клиент или non-default клиент.

Требования к решению:
1) Обучить модель бинарной классификации (можно взять baseline за основу).
2) Реализовать UI в любом формате:
   - сайт,
   - чат-бот,
   - Streamlit/Gradio,
   - Telegram-бот,
   - другой вариант.
3) В UI реализовать ввод признаков и вывод результата скоринга.

Желательно:
- возвращать вероятность дефолта (P(default));
- показывать метрики качества (ROC-AUC, PR-AUC, Accuracy);
- подобрать порог и описать компромисс между долей одобрений и риском.

Дополнительно:
- data/data_dictionary.csv — описание полей;
- data/correlations_with_target.csv — корреляции признаков с target.

Опционально (бонус):
Можно улучшить качество baseline-модели и обосновать, почему улучшение работает.
