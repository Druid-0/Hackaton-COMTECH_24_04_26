# Хакатон: кредитный скоринг

Кейс: есть baseline скоринга, ваша задача — собрать готовое решение с интерфейсом для пользователя.

## Что есть в репозитории

- `data/credit_scoring_dataset.csv` — основной датасет (500 строк)
- `data/data_dictionary.csv` — описание полей
- `data/correlations_with_target.csv` — корреляции признаков с таргетом
- `baseline.ipynb` — базовый пример обучения модели (логистическая регрессия на `numpy`)

## Формат данных

В датасете:

- 8 числовых признаков
- `target`: `1 = default (bad)`, `0 = non-default (good)`

## Что нужно сделать

Собрать end-to-end решение кредитного скоринга:

1. Использовать baseline как отправную точку и обучить модель.
2. Сделать UI в любом формате:
   - веб-страница / мини-сайт,
   - чат-бот,
   - Streamlit/Gradio,
   - Telegram-бот,
   - любой другой удобный интерфейс.
3. UI должен принимать значения всех признаков заявки и возвращать результат:
   - клиент дефолтный (`default`) или нет (`non-default`),
   - желательно с вероятностью (`P(default)`).

## Минимальные требования к результату

- Рабочий интерфейс для ввода признаков.
- Вызов обученной модели из UI.
- Ответ по заявке: `default / non-default`.
- Краткая инструкция запуска проекта.

## Можно улучшить (по желанию)

- Повысить качество модели (feature engineering, другой алгоритм, калибровка).
- Подобрать оптимальный порог классификации.
- Показать метрики (ROC-AUC, PR-AUC, Accuracy) и объяснить выбор.
- Добавить простую интерпретацию результата (какие признаки повлияли сильнее).
